# Documentation Structure

This directory contains all project documentation organized into three categories:

## üìÅ Directory Structure

### `/process` - Development Process & Troubleshooting
Contains root cause analyses, fix attempts, deployment logs, and hotfix documentation. These files track the development process, bug fixes, and deployment history.

**Key Files:**
- Root Cause Analyses (RCA): `BUTTON-PERMISSION-RCA.md`, `CAMERA-RCA.md`
- Fix Attempts: `FIX-ATTEMPTS.md`, `FIX-SUMMARY.md`
- Hotfixes: `HOTFIX-V2.1.md`, `HOTFIX-NOTES.md`
- Deployment Logs: `DEPLOYED.md`, `DEPLOYED-V2.md`, `DEPLOYED-ENHANCED.md`
- Field Testing: `FIELD-TEST-STATUS.md`

### `/documentation` - User & Developer Documentation
Contains user-facing guides, API documentation, and requirements. These files are meant for end users and developers using the application.

**Key Files:**
- Main README: `README.md` (project overview)
- Feature Guides: `DEMO-MODE-GUIDE.md`, `IMU-HEADING-GUIDE.md`
- Requirements: `UI-V2-REQUIREMENTS.md`
- Version-specific: `README-UNIFIED.md`

### `/learning` - Implementation Reference & Tutorials
Contains implementation summaries, build guides, and learning resources. These files document how features were built and serve as reference material.

**Key Files:**
- Build Summaries: `BUILD-SUMMARY.md`, `IMPLEMENTATION-SUMMARY.md`
- Feature Implementations: `IMU-IMPLEMENTATION.md`, `UI-V2-IMPLEMENTATION-SUMMARY.md`
- Performance: `PERFORMANCE-V2.md`
- Implementation Plans: `IMU-IMPLEMENTATION-PLAN.md`, `IMU-FIX-PLAN.md`
