## A. ABSTRACT / PROJECT SUMMARY (150-200 words)

Sightline is a **B2B SaaS platform** that enables museums and heritage sites to launch immersive AR guides in **30 days**, cutting deployment costs by 80% compared to traditional bespoke solutions.

**Problem:** Museums face a "digital deadlock": custom AR apps cost HK$500k+ and take 6+ months to build, making them inaccessible for most exhibitions. Meanwhile, 70% of visitors disengage with static text labels.

**Solution:** A modular "Content-as-a-Service" platform. Venues pay a setup fee to digitize their space (Digital Twin), then use a self-serve CMS to update AR content instantly. The "Hybrid Anchoring" engine delivers on-object content outdoors (GPS) and indoors (Vision AI) without complex hardware.

**Innovation:** Shift from "One-Off Project" to "Modular Platform." Includes a "Dynamic Q&A Loop" where AI tailors content to the visitor (e.g., Child vs. Historian).

**Market:** 60+ HK museums & 500+ schools.

**6-Month Goal:** Prove the "30-Day Deployment" model with 1 paid pilot at a major HK venue (e.g., Science Museum/Disneyland).

### Creative Solutions

**1. Leader Line Visualization**

**Problem:** Users lose track of which AR card refers to which object (occlusion, clutter).

**Solution:** Dynamic SVG line from anchor point to card, with:
- Edge-snapping: Line follows building edges (using Streetscape Geometry)
- Depth-aware: Line fades when object is behind user
- Visual hierarchy: Thicker line = higher confidence lock

**2. Far-Field Bearing Billboard**

**Problem:** GPS accuracy degrades beyond 100m; can't anchor to distant landmarks (IFC @ 2km).

**Solution:** Bearing-based billboard with distance chip:
- Calculate bearing/elevation from user pose to landmark
- Render card in that direction (not world-locked)
- Display "IFC Â· 2.1 km" to set expectation
- Hand off to local anchoring when user approaches <100m

**3. Confidence Gating UI**

**Problem:** Binary lock/fail creates frustration (flickers, false positives).

**Solution:** Three-state system with clear user feedback:
- **Green:** Locked, show full card + enable interactions
- **Amber:** Uncertain, show 2 nearest POI chips, prompt "Which one?"
- **Grey:** Searching, show "Steady camera..." hint

## J. RISKS & MITIGATIONS

### Technical Risks

**Risk 1: GPS Drift Outdoors (Â±5-15m)**
- **Probability:** High (inherent to consumer GPS)
- **Impact:** Medium (anchor appears offset from landmark)
- **Mitigation:**
  - Confidence state machine shows amber when GPS uncertainty >10m
  - Vision AI fallback: User can tap "Re-lock with camera"
  - Far-field bearing mode for >100m distances (no precise anchor needed)
- **Early Test:** Field test at Clock Tower (Month 1) to measure actual drift; adjust radius thresholds

**Risk 2: Vision AI Latency Spikes (>2s)**
- **Probability:** Medium (Gemini API variability)
- **Impact:** Medium (user experience degradation)
- **Mitigation:**
  - Local caching: POI responses cached by 50m grid tiles (10-min TTL)
  - Parallel requests: Send GPS + Vision API calls simultaneously, use whichever returns first
  - Timeout handling: If API >3s, show "Re-aim to lock" prompt
- **Early Test:** Stress test API with 100 sample images (Month 1) to establish p90 latency baseline

**Risk 3: Quest 3 Passthrough Camera API Changes**
- **Probability:** Low-Medium (experimental API, OS v74+)
- **Impact:** High (indoor demo breaks)
- **Mitigation:**
  - Phone-first strategy: Quest 3 is enhancement, not MVP blocker
  - Meta's sample code provides stable baseline (tested by community)
  - Fallback: Use Vuforia Image Targets if Passthrough API deprecated
- **Early Test:** Build Quest 3 prototype in Month 2 (Week 1) to validate API stability
